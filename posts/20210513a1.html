重装Ubuntu系统踩坑记录

发表于 2021-05-13 更新于 2021-06-07

一. 详情

实验室服务器总出问题，打算重新做一下系统。

二. 安装步骤

  1. 下载系统ISO文件，创建启动盘，BIOS设置u盘启动等等略过。
  2. 服务器电脑是两个硬盘，而且里面的数据都不要了。所以在安装的时候，创建分区的时候，选择Something else，下一步窗口会看到原来系统的efi等文件，选中右键->delete。挨个删除，只剩下free space。然后进行分区，此处略（参考分区方法）。 注意这样的话，你硬盘里面的东西就会全部格式化，别忘了备份重要数据。
  3. 下面就是正常安装步骤，等待安装完成就好了。

三.踩坑记录

  1. 此次安装的系统是20.04，打算安装cuda 10.1，但是Ubuntu20.04默认带的是gcc版本为9.3，所以需要降级，安装gcc-7版本。

sudo apt-get install gcc-7 g++-7

但是事情肯定不是这么简单，我运行的时候报错，E: Unmet dependencies. Try apt --fix-broken install，我运气比较好，运行sudo apt --fix-broken install,顺利解决问题。如果运气不好，可以参考此链接。 接下来顺利安装gcc和g++对应版本，然后把系统默认的gcc改为7版本。

sudo update-alternatives –install /usr/bin/gcc gcc /usr/bin/gcc-7 9

sudo update-alternatives –install /usr/bin/gcc gcc /usr/bin/gcc-9 1
g++修改步骤同上，此处参考链接。

  1. 然后安装英伟达显卡驱动：使用ubuntu-drivers devices查看机器建议安装版本，然后安装就好了。
  2. (1)下一步安装cuda，因为是20.04系统安装10.1版本，但是英伟达官网没有对应20.04版本的cuda10.1，所以直接下载对应18.04版本的就好。根据官网命令进行安装。 注意因为你前面已经安装驱动了，所以到CUDA Installer 界面的时候，把Driver选项去掉，中括号里面的X去掉，不然后面会报错。 安装完会出现一个WARNING，不用管它，意思是没有安装驱动，因为我们前面已经安装了，所以没有问题。
    (2)安装完成之后打开家目录下的 ~/.bashrc文件，末尾添加如下内容： export PATH=/usr/local/cuda-10.1/bin${PATH:+:${PATH}} export LD_LIBRARY_PATH=/usr/local/cuda-10.1/lib64${LD_LIBRARY_PATH:+:${LD_LIBRARY_PATH}
    (3) 保存文件，然后更新环境:source ~/.bashrc。
  3. 下一步安装cuDNN，略。参考此链接。只需注意一点，解压的时候不要忘了使用参数-C。
  4. 至此，基本环境已经配置完成了。

四. 安装软件

  1. 因为服务器是多人使用的，所以安装软件要让所有人都可以使用。
  2. 首先安装anaconda，注意安装路径选择在/usr/local里面就行了，然后再/etc/profile里面添加环境变量。此处参考链接。然后source /etc/profile。
    （1）其他用户用的时候，要先conda init命令。这时候，多用户创建环境是互相不影响的，除了都能看到root用户下面的base环境以外，其余只有自己的，其余用户是看不到的。
    （2）而且每个人要单独换源，不知道为什么，自己只能管自己的。
    （3）每个人的环境在/home/你的用户名/.conda/文件夹下。
  3. 安装其他软件同理。
  4. 多挂载一块硬盘：
    （1）首先断电把硬盘插在主板上，然后进入root用户，使用df -hl命令，查看自己挂载的硬盘。如果这个命令不显示刚装的硬盘，那就使用fdisk命令，找到硬盘位置。
    （2）创建挂载地方文件夹，然后使用sudo mount /dev/硬盘名字 挂载位置。
    （3）挂载完成以后要设置自动启动，不然重启以后还是要再挂一次。首先查看sudo blkid 你硬盘的位置，会返回一个UUID信息，记住这个数据。然后执行sudo vim /etc/fstab打开配置文件，最后添加UUID=****** 硬盘位置 类型 defaults 0 2。运行sudo mount -a检查是否配置正确，没有输出就行。如果配置错误可能导致开不了机。
    （4）如果是已经使用的硬盘，想换地方，需要切断联系，然后再挂载，就行了。切不断就强制切断。
